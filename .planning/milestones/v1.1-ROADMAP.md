# Milestone v1.1: Site Visual Refresh

**Status:** SHIPPED 2026-02-11
**Phases:** 8-11
**Total Plans:** 4

## Overview

Transform the documentation site from default Starlight into a polished, branded pentesting toolkit with a dark + orange/amber theme, custom logo, redesigned homepage, and cleaned-up navigation.

## Phases

### Phase 8: Theme Foundation

**Goal**: Site displays a cohesive dark + orange/amber visual identity across all pages and components
**Depends on**: Nothing (foundation for all visual work)
**Requirements**: THEME-01, THEME-02, THEME-03
**Plans**: 1 plan

Plans:
- [x] 08-01-PLAN.md -- Orange/amber CSS custom property overrides and config registration

**Details:**
- Orange/amber accent palette applied to all UI accent elements (links, sidebar, tabs, buttons, focus rings, search highlights) in dark mode
- WCAG AA compliant amber accent values for light mode (accent-high at hsl(28, 85%, 28%) for >= 4.5:1 contrast vs white)
- Deeper dark backgrounds (hsl(220, 15%, 8%)) for a distinct hacker/terminal aesthetic
- Used only :root variable overrides (no bare element/class selectors) to safely override Starlight's @layer cascade

### Phase 9: Brand Identity

**Goal**: Site header and browser tab display a recognizable project-specific brand instead of generic Starlight defaults
**Depends on**: Phase 8 (logo colors must work against the themed header)
**Requirements**: BRAND-01, BRAND-02
**Plans**: 1 plan

Plans:
- [x] 09-01-PLAN.md -- Terminal-prompt SVG logo, favicon, and Astro config registration

**Details:**
- Terminal-prompt icon (rounded rectangle with >_ motif) as site brand identity
- Dual-variant logo: amber (#f5c97a) for dark mode, dark amber (#854d0e) for light mode
- Adaptive favicon with CSS prefers-color-scheme media query for OS-level theme matching
- Logo registered in Starlight config alongside existing "Networking Tools" title text

### Phase 10: Navigation Cleanup

**Goal**: Sidebar navigation shows only meaningful page links without redundant section headers
**Depends on**: Phase 8 (sidebar styling affected by theme)
**Requirements**: NAV-01
**Plans**: 1 plan

Plans:
- [x] 10-01-PLAN.md -- Add sidebar.hidden frontmatter to section index pages

**Details:**
- Removed redundant "Tools", "Guides", and "Diagnostics" links from their respective sidebar groups
- Used sidebar.hidden frontmatter (Starlight built-in) rather than astro.config.mjs excludes
- All index pages remain accessible via direct URL -- only hidden from sidebar navigation

### Phase 11: Homepage Redesign

**Goal**: Homepage serves as a compelling entry point that showcases the full toolkit and guides visitors to what they need
**Depends on**: Phase 8 (card styling requires theme), Phase 9 (hero uses logo image)
**Requirements**: BRAND-03, HOME-01, HOME-02, HOME-03
**Plans**: 1 plan

Plans:
- [x] 11-01-PLAN.md -- Homepage MDX rewrite with hero, feature cards, tool grids, and guide links

**Details:**
- Replaced minimal markdown homepage with full MDX landing page using Starlight Card/CardGrid/LinkCard components
- Hero section displays project logo (dark/light variants) with tagline "Pentesting scripts and diagnostics. Ready to run."
- All 17 tools organized into Security Tools (13) and Networking Tools (4) card grids
- 3 diagnostic pages and 6 guide/learning path pages linked via dedicated card grid sections
- Homepage section spacing CSS scoped to splash pages via [data-has-hero] selector

## Milestone Summary

**Key Decisions:**
- Use only :root CSS variable overrides (no element/class selectors) for safe Starlight theme override
- Light mode accent-high at hsl(28,85%,28%) for WCAG AA contrast against white
- Terminal prompt >_ motif as brand icon -- stroke-only SVG readable at 16px
- Logo accompanies title text (replacesTitle: false) since icon has no text content
- Use sidebar.hidden frontmatter (not astro.config.mjs exclude) to remove redundant index entries
- Convert index.md to index.mdx (git mv) to enable Starlight component imports
- Use [data-has-hero] CSS scoping for splash-page-only section spacing
