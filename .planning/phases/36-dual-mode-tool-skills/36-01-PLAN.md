<!-- Dual-Mode SKILL.md Template (from 36-RESEARCH.md Pattern 1) -->
<!-- Every pilot skill MUST follow this structure exactly -->

```yaml
---
name: <tool>
description: >-
  <Action verb> ... with <tool>. <Keyword list>. (under 200 chars)
disable-model-invocation: true
---
```

Required sections in order:
1. `# <Tool Name>` -- one-line summary
2. `## Tool Status` -- dynamic injection for tool install + wrapper detection
3. `## Mode: Wrapper Scripts Available` -- wrapper script commands with -j -x
4. `## Mode: Standalone (Direct Commands)` -- inline tool commands grouped by use case
5. `## Defaults` -- target defaults, common flags
6. `## Target Validation` -- scope reference

Dynamic injection patterns:
- Tool installed: `!` `command -v <binary> > /dev/null 2>&1 && echo "YES -- $(<version_cmd>)" || echo "NO -- Install: <install_hint>"`
- Wrapper scripts: `!` `test -f scripts/<tool>/<representative_script>.sh && echo "YES -- use wrapper scripts for structured JSON output" || echo "NO -- using standalone mode with direct commands"`

Install hints (from check-tools.sh):
- dig: `apt install dnsutils (Debian/Ubuntu) | brew install bind (macOS)`
- curl: `apt install curl (Debian/Ubuntu) | brew install curl (macOS)`
- nc: `apt install netcat-openbsd (Debian/Ubuntu) | brew install netcat (macOS)`

Version commands:
- dig: `dig -v 2>&1 | head -1`
- curl: `curl --version 2>/dev/null | head -1`
- nc: `nc -h 2>&1 | head -1 || true`
