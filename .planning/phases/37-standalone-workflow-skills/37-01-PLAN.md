**Recon workflow (6 steps):**

Add `## Environment Detection` section after the Target section:
```
## Environment Detection

- Wrapper scripts available: !`test -f scripts/nmap/discover-live-hosts.sh && echo "YES" || echo "NO"`
```

Transform each of the 6 steps to have per-step dual-mode branching:

Step 1 (Host Discovery) -- standalone commands from nmap tool skill (Host Discovery section):
- `nmap -sn $ARGUMENTS/24` -- Ping sweep to find live hosts
- `nmap -PS22,80,443 $ARGUMENTS/24` -- TCP SYN discovery on common ports
- `nmap -sn -PR $ARGUMENTS/24` -- ARP scan (local network only, fastest)

Step 2 (Port Scanning) -- standalone commands from nmap tool skill (Port Scanning section):
- `nmap -sS -sV $ARGUMENTS` -- TCP SYN scan with service versions
- `nmap -p- $ARGUMENTS` -- Scan all 65535 ports
- `nmap -A $ARGUMENTS` -- Aggressive scan (OS, versions, scripts, traceroute)

Step 3 (DNS Records) -- standalone commands from dig tool skill (DNS Records section):
- `dig $ARGUMENTS A +noall +answer` -- IPv4 address
- `dig $ARGUMENTS MX +noall +answer` -- Mail exchange servers
- `dig $ARGUMENTS NS +noall +answer` -- Authoritative nameservers
- `dig $ARGUMENTS TXT +noall +answer` -- SPF, DKIM records

Step 4 (Zone Transfer) -- standalone commands from dig tool skill (Zone Transfers section):
- `dig $ARGUMENTS NS +short` -- Find authoritative nameservers first
- `dig axfr $ARGUMENTS @<nameserver>` -- Attempt AXFR zone transfer

Step 5 (SSL/TLS) -- standalone commands from curl tool skill (SSL/TLS section):
- `curl -vI https://$ARGUMENTS 2>&1 | grep -E "SSL|subject|issuer|expire"` -- Certificate details
- `curl --tlsv1.2 -sI https://$ARGUMENTS` -- Test TLS 1.2 support
- `curl --tlsv1.3 -sI https://$ARGUMENTS` -- Test TLS 1.3 support

Step 6 (Subdomains) -- standalone commands from gobuster tool skill (Subdomain section):
- `gobuster dns -d $ARGUMENTS -w subdomains.txt` -- DNS subdomain brute-force
- `gobuster dns -d $ARGUMENTS -w subdomains.txt -t 20` -- 20 concurrent threads

Add educational context (1-2 sentences) explaining WHY each step matters, before the branching commands. Keep the existing educational text where present, add where missing.

- "**If standalone:** Review the command output directly for key findings."
- Keep the bullet points about noting key findings, skipping missing tools, and adapting steps.

Keep the Summary section unchanged (already mode-agnostic).

Transform each of the 5 steps to have per-step dual-mode branching:

Step 1 (Hash ID) -- standalone commands from john tool skill (Hash Identification section):
- `john --list=formats | grep -i <suspected_type>` -- Filter formats by type
- `john --format=raw-md5 --test hash.txt` -- Test specific format
- `hashcat --identify hash.txt` -- Alternative: hashcat's identify mode (if hashcat 6.2.6+)

Step 2 (NTLM) -- standalone commands from hashcat tool skill:
- `hashcat -m 1000 -a 0 $ARGUMENTS wordlist.txt` -- Dictionary attack on NTLM
- `hashcat -m 1000 -a 3 $ARGUMENTS ?a?a?a?a?a?a` -- Mask attack (6 chars)

Step 3 (Web Hashes) -- standalone commands from hashcat tool skill:
- `hashcat -m 0 -a 0 $ARGUMENTS wordlist.txt` -- MD5 dictionary attack

- `hashcat -m 1400 -a 0 $ARGUMENTS wordlist.txt` -- SHA-256
- `hashcat -m 3200 -a 0 $ARGUMENTS wordlist.txt` -- bcrypt

Step 4 (Linux) -- standalone commands from john tool skill (Linux section):
- `unshadow /etc/passwd /etc/shadow > combined.txt` -- Merge passwd and shadow
- `john --wordlist=wordlist.txt combined.txt` -- Dictionary attack
- `john --wordlist=wordlist.txt --rules combined.txt` -- With rule mutations
- `john --show combined.txt` -- Display cracked passwords

Step 5 (Archives) -- standalone commands from john tool skill (Archive section):
- `zip2john protected.zip > hash.txt` -- Extract hash from ZIP
- `rar2john protected.rar > hash.txt` -- Extract from RAR
- `john --wordlist=wordlist.txt hash.txt` -- Crack extracted hash
