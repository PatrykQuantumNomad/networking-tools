
Per-step standalone commands (source: corresponding tool skills):

Step 1 (Port and Service Scan) -- from nmap tool skill (Port Scanning):
- `nmap -sS -sV $ARGUMENTS` -- TCP SYN scan with service version detection
- `nmap -p- $ARGUMENTS` -- Scan all 65535 ports
- `nmap -A $ARGUMENTS` -- Aggressive scan (OS, versions, scripts, traceroute)

Step 2 (Web Vuln NSE) -- from nmap tool skill (NSE Scripts):
- `nmap --script=http-vuln-* $ARGUMENTS` -- Run all HTTP vulnerability scripts
- `nmap --script=ssl-enum-ciphers -p 443 $ARGUMENTS` -- Enumerate SSL/TLS ciphers
- `nmap -sV --script=vulners $ARGUMENTS` -- Check service versions against CVE database

Step 3 (Web Server Analysis) -- from nikto tool skill:
- `nikto -h $ARGUMENTS` -- Default scan for known vulnerabilities
- `nikto -h $ARGUMENTS -Tuning 123` -- Focus on file upload, default files, info disclosure
- `nikto -h $ARGUMENTS -ssl` -- Force SSL mode for HTTPS targets

Step 4 (SQL Injection) -- from sqlmap tool skill:
- `sqlmap -u "$ARGUMENTS" --batch` -- Auto-detect and test SQL injection
- `sqlmap -u "$ARGUMENTS" --batch --level=5 --risk=3` -- Maximum detection coverage
- `sqlmap -u "$ARGUMENTS" --batch --dbs` -- Enumerate databases if injectable

Step 5 (HTTP Endpoints) -- from curl tool skill:
- `curl -I $ARGUMENTS` -- Check response headers
- `curl -X OPTIONS $ARGUMENTS -i` -- Test allowed HTTP methods
- `curl -sI $ARGUMENTS | grep -iE "x-frame|x-content|strict-transport|content-security"` -- Check security headers

Step 1 (DNS Diagnostics) -- standalone uses raw dig commands (replaces scripts/diagnostics/dns.sh):
- `dig $ARGUMENTS A +noall +answer` -- A record resolution
- `dig $ARGUMENTS AAAA +noall +answer` -- IPv6 resolution
- `dig $ARGUMENTS MX +noall +answer` -- Mail exchange records
- `dig $ARGUMENTS NS +noall +answer` -- Authoritative nameservers
- `dig @8.8.8.8 $ARGUMENTS A +short` -- Google DNS propagation check
- `dig @1.1.1.1 $ARGUMENTS A +short` -- Cloudflare DNS propagation check
- `dig -x $(dig $ARGUMENTS A +short | head -1) +short` -- Reverse DNS lookup
Note in the standalone section: "Report findings as PASS/FAIL for each check."

Step 2 (Connectivity) -- standalone uses raw ping/curl (replaces scripts/diagnostics/connectivity.sh):
- `ping -c 4 $ARGUMENTS` -- ICMP reachability test
- `curl -sI --connect-timeout 5 http://$ARGUMENTS` -- HTTP connectivity check
- `curl -sI --connect-timeout 5 https://$ARGUMENTS` -- HTTPS connectivity check
- `nc -zv $ARGUMENTS 22 2>&1` -- SSH port check
- `nc -zv $ARGUMENTS 80 2>&1` -- HTTP port check
- `nc -zv $ARGUMENTS 443 2>&1` -- HTTPS port check
Note: "Report each check as PASS (reachable) or FAIL (unreachable)."

Step 3 (Performance) -- standalone uses raw ping/curl (replaces scripts/diagnostics/performance.sh):
- `ping -c 10 $ARGUMENTS` -- Latency stats (min/avg/max/stddev)
- `curl -o /dev/null -s -w "DNS: %{time_namelookup}s\nConnect: %{time_connect}s\nTLS: %{time_appconnect}s\nTotal: %{time_total}s\n" https://$ARGUMENTS` -- HTTP timing breakdown
Note: "Review ping statistics for packet loss and jitter. Review curl timings for DNS/connect/TLS overhead."

Step 4 (Network Path Tracing) -- standalone from traceroute tool skill:
- `traceroute $ARGUMENTS` -- Standard UDP traceroute
- `mtr --report $ARGUMENTS` -- Combined traceroute+ping report
- `traceroute -T $ARGUMENTS` -- TCP traceroute (bypasses ICMP filters)

Step 5 (DNS Propagation) -- standalone from dig tool skill (Propagation section):
- `dig @8.8.8.8 $ARGUMENTS A +short` -- Google DNS
- `dig @1.1.1.1 $ARGUMENTS A +short` -- Cloudflare DNS
- `dig @208.67.222.222 $ARGUMENTS A +short` -- OpenDNS
- `dig @9.9.9.9 $ARGUMENTS A +short` -- Quad9 DNS
Note: "Compare responses across resolvers. Matching results indicate complete propagation."
