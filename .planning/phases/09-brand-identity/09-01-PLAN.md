  <action>
Create three SVG files using a terminal prompt (`>_`) motif inside a rounded rectangle, evoking CLI/terminal tools. The design must be simple geometric shapes -- no fine detail -- readable at 16px (favicon) through 40px (header logo).

**Design spec -- terminal prompt icon:**
- Rounded rectangle "terminal window" outline (rx="3" or similar for subtle rounding)
- Chevron (`>`) on the left side
- Underscore cursor (`_`) to the right of the chevron
- viewBox="0 0 32 32" (square aspect ratio)
- No text elements (text renders poorly at small sizes in SVG-as-img)
- Transparent background (no background fill -- the page background shows through)
- Strokes should be at least 2px wide for visibility at small sizes

**logo-dark.svg** (for dark mode header -- dark background):
- Fill/stroke color: `#f5c97a` (matches --sl-color-accent-high dark mode: hsl(38, 95%, 82%))
- This is amber/golden on a transparent background, visible against the dark header (hsl(220, 15%, 8%))

**logo-light.svg** (for light mode header -- light background):
- Fill/stroke color: `#854d0e` (matches --sl-color-accent-high light mode: hsl(28, 85%, 28%))
- This is dark burnt amber on transparent, visible against white/near-white header background
- Same paths/shapes as logo-dark.svg, only the color differs

**favicon.svg** (replaces existing default in site/public/):
- Same terminal prompt icon shape as the logo files
- Uses CSS classes + embedded `<style>` block with `@media (prefers-color-scheme: dark)` for OS-level adaptation
- Light mode default: `.primary { fill: #854d0e; }` (dark amber)
- Dark mode media query: `.primary { fill: #f5c97a; }` (amber/golden)
- viewBox="0 0 32 32"

IMPORTANT: The three SVG files must use the same icon shape/paths. Only the fill colors and the color-switching mechanism differ (static fills for logo files, media query for favicon).

<task type="auto">

  <files>
    site/astro.config.mjs

  <action>
Add the `logo` configuration to the `starlight()` config in `site/astro.config.mjs`:

```javascript
logo: {
  dark: './src/assets/logo-dark.svg',
  light: './src/assets/logo-light.svg',
  alt: 'Networking Tools',
},
```

Place the `logo` property after `description` and before `customCss` in the starlight config object.

Do NOT set `replacesTitle: true` -- the default (false) keeps the "Networking Tools" text visible alongside the logo icon, which is correct since the logo is a graphical icon without text.

Do NOT change the `favicon` config -- Starlight defaults to `/favicon.svg` which is already the correct path for the replaced file.

After updating the config, run `npm run build` from the `site/` directory to verify the site builds successfully with the new logo assets.
