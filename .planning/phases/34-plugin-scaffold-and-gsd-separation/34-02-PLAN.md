<task type="auto">
  <name>Task 1: Create plugin boundary validation script</name>
  <files>scripts/validate-plugin-boundary.sh</files>
  <action>
Create `scripts/validate-plugin-boundary.sh` -- an allowlist-based boundary enforcement script. Place it at the repo root level `scripts/` (NOT inside netsec-skills/).

The script must:
1. Accept optional argument for plugin directory path (default: `netsec-skills`)
2. Use `set -euo pipefail` for strict mode
3. Find all files (regular files and symlinks) in the plugin directory
4. Check each file against the ALLOWLIST from CONTEXT.md:
   - `skills/**/*.md` -- Skill markdown files (including nested in category subdirs)
   - `hooks/*.sh` -- Hook shell scripts
   - `hooks/*.json` -- Hook configuration
   - `agents/*.md` -- Agent definitions
   - `scripts/*.sh` -- Utility scripts
   - `.claude-plugin/plugin.json` -- Plugin manifest
   - `marketplace.json` -- Content catalog
   - `README.md` -- Documentation
5. Any file NOT matching the allowlist is a VIOLATION -- print it and increment counter
6. Separately scan for any `gsd-*` prefixed files (backup check even though allowlist should catch them)
7. Check for broken symlinks -- any symlink that does not resolve is a VIOLATION
8. Validate JSON syntax on all .json files using `jq . file > /dev/null 2>&1`
9. Print summary: total files checked, violations found, broken symlinks
10. Exit 0 if clean, exit 1 if any violations

Use the Pattern 5 from 34-RESEARCH.md as a starting point but enhance it with:
- Broken symlink detection
- JSON syntax validation
- File count summary
- Clear PASS/FAIL output with counts

Add a standard script header comment block with description, usage, and examples. Do NOT add the `@description/@usage/@dependencies` metadata header format used by the pentesting scripts -- this is a repo utility script, not a tool wrapper.

Make the script executable: `chmod +x scripts/validate-plugin-boundary.sh`
  </action>
