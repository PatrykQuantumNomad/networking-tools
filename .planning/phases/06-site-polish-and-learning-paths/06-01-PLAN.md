    For each of the 15 tool .md files in site/src/content/docs/tools/:

    1. Rename from .md to .mdx using `git mv` (preserves git history)
    2. Add the import statement immediately after the frontmatter closing `---`:
       ```
       import { Tabs, TabItem } from '@astrojs/starlight/components';
       ```
    3. Find the existing "## Install" section (if present as a table) and replace it with a Tabs component. If no Install section exists, add one after "## What It Does" or "## Running the Examples Script".
    4. Remove any `badge` frontmatter with `text: 'New'` (dig and traceroute have these -- these tools are no longer new).

    Install tab pattern for each tool (use `syncKey="os"` on all, `icon="apple"` for macOS, `icon="linux"` for Linux tabs):

    ```mdx
    ## Install

    <Tabs syncKey="os">
      <TabItem label="macOS" icon="apple">

        ```bash
        brew install <package>
        ```

      </TabItem>
      <TabItem label="Debian / Ubuntu" icon="linux">

        ```bash
        sudo apt install <package>
        ```

      </TabItem>
      <TabItem label="RHEL / Fedora" icon="linux">

        ```bash
        sudo dnf install <package>
        ```

      </TabItem>
    </Tabs>
    ```

    CRITICAL whitespace rules for MDX: blank line after opening `<TabItem>` tag and before closing `</TabItem>` tag. Code blocks inside tabs must have surrounding blank lines.

    Install data per tool (from research):
    - nmap: brew install nmap / apt install nmap / dnf install nmap
    - tshark: brew install wireshark / apt install tshark / dnf install wireshark-cli
    - metasploit: Special case -- use TWO tabs: "macOS / Linux" (nightly installer link: https://docs.metasploit.com/docs/using-metasploit/getting-started/nightly-installers.html) and "Kali Linux" (pre-installed). No standard brew/apt install.
    - aircrack-ng: brew install aircrack-ng / apt install aircrack-ng / dnf install aircrack-ng
    - hashcat: brew install hashcat / apt install hashcat / dnf install hashcat
    - skipfish: Special case -- only TWO tabs: "macOS (MacPorts)" with `sudo port install skipfish` (note: not in Homebrew) and "Debian / Ubuntu" with `sudo apt install skipfish`. No RHEL/Fedora tab (unavailable).
    - sqlmap: brew install sqlmap / apt install sqlmap / dnf install sqlmap
    - hping3: brew install draftbrew/tap/hping / apt install hping3 / dnf install hping3 (note the special Homebrew tap)
    - john: brew install john / apt install john / dnf install john
    - nikto: brew install nikto / apt install nikto / dnf install nikto
    - foremost: brew install foremost / apt install foremost / dnf install foremost
    - dig: brew install bind / apt install dnsutils / dnf install bind-utils
    - curl: macOS tab says "Pre-installed on macOS" (no brew command) / apt install curl / dnf install curl
    - netcat: brew install netcat / apt install netcat-openbsd / dnf install nmap-ncat
    - traceroute: macOS tab says "Pre-installed on macOS" (add note: mtr via `brew install mtr`) / apt install traceroute (add: `sudo apt install mtr`) / dnf install traceroute (add: `sudo dnf install mtr`)

    If the existing page already has an Install section as a markdown table, replace that table with the Tabs component. Keep the "## Install" heading.
